## 🧰 设计原则

定义应用协议时，应当遵循以下核心设计原则：

1. **约定大于配置**：减少明确的配置需求，优先使用合理的默认约定
2. **职责单一**：每个协议应专注于单一功能或目的，避免过度复杂
3. **最小可行产品(MVP)**：从最核心功能开始，确保基础用例能够正常工作
4. **奥卡姆剃刀原则**：在同等条件下，应选择最简单的设计方案
5. **一致性**：保持与其他协议的设计风格和使用模式一致

这些原则有助于确保协议设计的简洁性、可用性和可维护性。
